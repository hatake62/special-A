# 定数畳み込みの説明資料

## 定数畳み込みとは

定数畳み込みとは、実行時に計算しなくても結果が分かる式を、コンパイル時にあらかじめ計算しておく最適化である。

例えば `10 * 20 + 30 * 40 - 50` は、変数や入力値に依存しない。プログラムを実行するたびに VM 上で `10 * 20`、`30 * 40`、加算、減算を行わなくても、コンパイル時に `1350` という値へ置き換えられる。

この最適化の目的は、プログラムの意味を変えずに、実行時に処理する VM 命令数を減らすことである。特に、ループの中で同じ定数式が何度も評価される場合、削減した命令数が反復回数分だけ積み上がるため、実行時間にも効果が出やすい。

## 最適化前のコード例

今回のベンチマークでは、配列を走査する関数の中に定数式を含めている。

```python
def score_values(values):
    i = 0
    total = 0
    while i < len(values):
        total = total + values[i] + (10 * 20 + 30 * 40 - 50)
        i = i + 1
    return total

arr = [1, 2, 3, 4, 5, 6, 7, 8]
i = 0
total = 0
while i < 5000:
    total = total + score_values(arr)
    i = i + 1
```

最適化前は、`score_values` のループを1回回るたびに、`10 * 20 + 30 * 40 - 50` を VM 命令として評価する。配列要素は8個あり、外側のループで `score_values` を5000回呼び出すため、この定数式は合計 `8 * 5000 = 40000` 回評価される。

## 最適化後のコード例

定数畳み込みを行うと、定数式はあらかじめ計算された値に置き換わる。

```python
def score_values(values):
    i = 0
    total = 0
    while i < len(values):
        total = total + values[i] + 1350
        i = i + 1
    return total

arr = [1, 2, 3, 4, 5, 6, 7, 8]
i = 0
total = 0
while i < 5000:
    total = total + score_values(arr)
    i = i + 1
```

この変換では、プログラムの計算結果は変わらない。変わるのは、実行時に VM が処理する命令列である。最適化後は、毎回のループで複数の算術命令を実行せず、`1350` をスタックに積むだけでよくなる。

## VM命令列の変化

最適化前の定数式 `(10 * 20 + 30 * 40 - 50)` は、概念的には次のような VM 命令列になる。

```text
push 10
push 20
mul
push 30
push 40
mul
add
push 50
sub
```

この部分だけで9命令ある。これに対して、定数畳み込み後は次の1命令で表現できる。

```text
push 1350
```

つまり、定数式1回あたり `9 - 1 = 8` 命令を削減できる。今回のベンチマークでは、この式が配列8要素分、外側ループ5000回分だけ実行されるため、削減量は次のようになる。

```text
8命令 * 8要素 * 5000回 = 320000命令
```

このように、定数畳み込みは単体では小さな変換だが、ホットパス上で繰り返し実行される式に適用されると、VM命令数を大きく減らせる。

## ベンチマーク結果

今回の「拡張VM 定数畳み込み・配列」ベンチマークでは、定数畳み込みによって VM 命令数が `1165020` から `845020` に減った。差分は `320000` 命令である。

平均実行時間も `1.248271` 秒から `0.901370` 秒に短縮された。短縮時間は `0.346901` 秒で、約 `27.8%` の改善である。命令数の削減率も約 `27.5%` であり、VM が実行する命令を減らした効果が実行時間にもほぼ同じ傾向で現れている。

| 項目 | 最適化前 | 最適化後 | 改善 |
| --- | ---: | ---: | ---: |
| VM命令数 | 1165020 | 845020 | 320000命令削減 |
| 平均実行時間 | 1.248271秒 | 0.901370秒 | 0.346901秒短縮 |

## 発表スライド用まとめ

定数畳み込みは、実行時に計算しなくても結果が分かる式を、コンパイル時に計算済みの値へ置き換える最適化である。今回の例では、ループ内の `(10 * 20 + 30 * 40 - 50)` を `1350` に置き換えた。

最適化前は、この定数式を評価するために `push` と算術命令を合わせて9命令実行していた。最適化後は `push 1350` の1命令で済むため、定数式1回あたり8命令を削減できる。この式は配列8要素分、外側ループ5000回分だけ実行されるため、合計で `320000` 命令の削減につながった。

ベンチマークでは、VM命令数が `1165020` から `845020` に減少し、平均実行時間も `1.248271` 秒から `0.901370` 秒へ短縮された。この結果から、定数畳み込みは特にループ内のような実行頻度の高い場所で効果が大きいことが分かる。

重要なのは、最適化によってプログラムの結果は変えず、実行時に VM が処理する命令だけを減らしている点である。今回の結果は、コンパイル時にできる計算を先に済ませることで、VM の実行コストを明確に下げられることを示している。
