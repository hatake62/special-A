自作言語処理系 発表

Special-A

Python風の小さな言語を、自作コンパイラと自作VMで実行する。さらに、最適化で速さと消費エネルギーを改善する処理系。

10分発表 + デモ

0から説明

何を作ったか

  • 入力は Python の一部に似たソースコード
  • 関数、配列、while、if、四則演算、比較を実行できる
  • try / except / raise を持つ拡張VMも実装
  • 定数畳み込みと関数インライン展開で最適化できる

最低ラインへの対応

関数: def / call / return

配列: リテラル / 添字アクセス / 添字代入 / len

前のレポートに依存せず、この処理系単体として説明

言語仕様

書けるプログラム

def sum_array(values):
    i = 0
    total = 0
    while i < len(values):
        total = total + values[i]
        i = i + 1
    return total

numbers = [10, 20, 30]
numbers[1] = 25
print(sum_array(numbers))
  • Pythonそのものではなく、Python風の小さな言語として扱う
  • 構文解析には Python 標準の ast を使い、内部用ASTへ変換
  • その後、自作VM命令へコンパイルして実行する
examples/array.src

実装方式

処理の流れ

1. ソースPython風のプログラムを書く
2. 構文解析Python ast から独自ASTへ変換
3. 最適化定数畳み込み、インライン展開
4. コンパイルスタックVM命令列を生成
5. 実行VMが命令を解釈して結果を出す
src/parser.py, src/optimizer.py, src/compiler.py, src/vm.py

VM

スタックで実行する

# 例: 10 + 20
push 10
push 20
add

# 配列
push 10
push 20
push 30
build_array 3
get_index
set_index
  • 値はスタックに積む
  • add / sub / mul / div は上の2値を取り出して計算
  • 関数呼び出しではフレームを積む
  • 配列操作は build_array / get_index / set_index 命令で実行
VM命令数を数えるので、最適化の効果を説明しやすい

凄さ 1

コンパイル時に計算を消す

# 最適化前
total = total + values[i] + (10 * 20 + 30 * 40 - 50)

# 定数畳み込み後
total = total + values[i] + 1350
  • 毎回同じ答えになる式を、実行前に計算しておく
  • ループの中にある定数式ほど効果が大きい
  • 0除算のように意味が変わる可能性がある式は畳み込まない
定数畳み込み: constant folding

凄さ 2

小さい関数呼び出しを消す

def add_const(x):
    return x + 1350

total = total + add_const(arr[i])
# インライン展開後
total = total + (arr[i] + 1350)
単純な return だけの関数を呼び出しなしの式へ置換
